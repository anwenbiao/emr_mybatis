全部实例

  • 设计模式--电子病历设计器(扩展toolbar)
    建议给病历模板设计者（开发人员，或者科主任）使用，可用来设计电子病历模板。也可以当做电子病历编辑器使用，此时输入的值可利用SDE对象暴露出的接口获取。增加自定义toolbar标题功能，可支持多语言
  • 编辑模式--电子病历设计器
    建议给医生使用，此时医生可以在原有模板中添加已有的控件，也可以给科主任用来设计模板。亦可通过SDE对象暴露出来的接口获取数据。
  • 只读模式--电子病历设计器
    建议该模式给医生查看使用，在该模式下电子病历中所有控件均不可编辑。
  • 按钮控制--电子病历设计器
    按钮通过事件控制！
  • 设计模式--电子病历编辑器
    建议给医生使用，此模式下医生可以编辑控件中的值，并且可编译控件的内容。选中文字后有惊喜。
  • 编辑模式--电子病历编辑器
    建议给医生使用，此模式下医生只能编辑控件中的值，其余均不可修改。
  • 只读模式--电子病历编辑器
    该模式只允许查看，控件不可被编辑。
  • 按钮控制--电子病历编辑器
    按钮通过事件控制！
